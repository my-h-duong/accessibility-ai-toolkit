# Accessibility Principles

## Semantic HTML First

Prefer native HTML elements before introducing ARIA.

## Native Controls First

Use built-in browser behavior whenever possible.

## Visible Help Text

Instructions should be visible when users need them.

Avoid tooltip-only help text.

## Keyboard Parity

Anything available with a mouse must also be available with a keyboard.

## Error Recovery

Users should be able to identify, understand, and recover from errors.
