# Accessibility

This repository is organized around reusable accessibility knowledge and review assets.

## Artifact Types

- `principles/` explains durable guidance and decision-making.
- `patterns/` documents recurring implementation and review problems with preferred approaches.
- `workflows/` defines repeatable review processes.
- `checklists/` provides task- and lens-specific review prompts.
- `findings/` contains finding templates and example findings.
- `references/` defines the evidence model and trusted source categories used to support findings.

## Review Lenses

Design, development, and content are treated as review lenses rather than separate top-level domains.

When a distinction matters:

- Principles and patterns may note `Applies To` or `Discipline`.
- Checklists and workflows should carry most discipline-specific guidance.
